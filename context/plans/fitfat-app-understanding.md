# FitFat App Understanding Plan

## Goal
Capture a durable, code-backed understanding of the app before any implementation work.

## Tasks
T01 — Inspect app structure
- Read the bootstrap, router, models, database schema, and feature modules.
- Confirm the major user flows and persistence approach.
- Done when the app’s top-level purpose and feature areas are identified.

T02 — Draft current-state context
- Write `context/overview.md`, `context/context-map.md`, and `context/glossary.md`.
- Keep the notes current-state oriented and tied to code paths.
- Done when the docs reflect the observed architecture and domain terms.

T03 — Capture open questions
- Record product and naming ambiguities that need user clarification.
- Focus on scope, terminology, and intended priority of diet vs exercise vs dashboard.
- Done when the remaining unknowns are clearly enumerated.

T04 — Verify and clean up
- Re-read the generated context for consistency and remove duplication.
- Ensure the docs are concise, reviewable, and ready for future sessions.
- Done when the context files are coherent and linked to the current codebase.
